# Domain unit tests

This folder is reserved for unit tests that verify domain behavior.

Use this folder for tests related to:

- entities;
- value objects;
- domain services;
- domain invariants;
- domain validation rules;
- domain-specific calculations.

Examples of future test areas:

- wallets;
- money;
- transactions;
- budgets;
- categories;
- financial goals.

Do not add tests here until the corresponding domain model is implemented and ready to be tested.
