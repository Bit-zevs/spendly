# Transactions

This folder is reserved for transaction-related domain model.

Possible future contents:

- transaction aggregate or entity;
- transaction identifier;
- transaction type;
- transaction status;
- transaction amount rules;
- transaction date rules;
- transaction-related domain errors.

Transactions represent money movement in Spendly.

Examples may include expenses, income, transfers, subscription charges, and corrections.

This folder should contain business rules only, not API endpoints, database queries, or infrastructure integrations.
